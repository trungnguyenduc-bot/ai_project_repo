Contribution Guidelines

Thank you for contributing to this Python AI project! Please start by creating a new branch from main for each feature or bug fix. Use the format feature/your-feature-name or fix/issue-description. Before submitting, ensure all code passes linting (flake8) and tests (pytest). Follow PEP 8 styling and include clear docstrings for functions and classes.

Commit messages should be concise and descriptive, using the convention:
type(scope): short summary
Examples:

feat(model): add neural network training module

fix(data): correct preprocessing bug

Submit a pull request (PR) to main with a summary of your changes and link related issues. All PRs require at least one review before merging.
